Main Paths

  • Onboarding: App -> BLE -> ESP32
  • Telemetry: ESP32 -> FastAPI -> Firestore
  • Live View: Firestore -> Client Apps
  • Alerts: FastAPI -> FCM -> Client Apps
  • Auth: Client Apps -> Firebase Auth

Layer 1: Presentation Layer

Flutter Mobile App

Android and iOS user interface for monitoring, setup, and configuration.

Flutter Web App (PWA)

Browser-based access to dashboards and system settings.

Flutter Desktop App

Windows client for local operations and data visualization.

Layer 2: Device Layer

BLE Provisioning

App sends WiFi credentials to ESP32 during onboarding and network changes.

ESP32 Controller

Collects sensor values, applies local control logic, and streams telemetry.

Sensors and Relays

Turbidity, pH, TDS, ultrasonic level plus pump/UV/bypass control outputs.

v v v

Layer 3: Application Layer

FastAPI Backend on Render

Receives WebSocket telemetry, validates payloads, and routes events.

Rule and Threshold Engine

Checks limits and generates warning/critical events for notifications.

v v v

Layer 4: Cloud Data Layer

Firebase Auth

Handles login, registration, and session identity for clients.

Cloud Firestore

Stores readings, rollups, thresholds, alerts, and configuration state.

Firebase Cloud Messaging

Delivers push notifications when backend rules detect threshold breaches.

Operational Data Flow

  1. 1 User Auth User signs in through Firebase Auth from any client app.
  2. 2 Device Setup BLE provisioning sends network credentials to ESP32.
  3. 3 Telemetry Ingestion ESP32 transmits sensor JSON to FastAPI over WebSocket.
  4. 4 Persist and Sync Backend validates and stores data in Firestore for app dashboards.
  5. 5 Alert Delivery Backend triggers FCM push notifications when limits are exceeded.

Figure: AGOS Layered System Framework. This framework separates user experience, hardware acquisition, backend processing, and cloud services to support reliable real-time monitoring and scalable deployment.
