⚡ AGOS — Breadboard Wiring Diagram (Full Build)

All 5 sensors + ESP32 NodeMCU 38-pin with resistors on breadboard

5V power (red wire)
3.3V power (orange — TDS only)
Ground (black/dark wire)
TRIG/ECHO → GPIO5/18 (purple)
Turbidity SIG → R1→GPIO34 (green)
pH PO → GPIO35 (pink)
TDS A → GPIO32 (blue)
Relay IN → GPIO26 (yellow)
Resistors (brown body = carbon film)
⚠ Turbidity Sensor (SEN0189) Powers from 5V. Its SIG output can reach 4.5V → will damage ESP32!
Must use the 3 resistors (10kΩ+10kΩ+10kΩ) divider on the breadboard before connecting to GPIO34. Wire path: SIG → R1 → [junction → GPIO34] and [R2a → R2b → GND].
⚠ JSN-SR04T ECHO pin ECHO output is 5V → will damage ESP32!
Must use 2 resistors (1kΩ+1kΩ) divider on breadboard before connecting to GPIO18. Wire path: ECHO → R3 → [junction → GPIO18] and [R4 → GND].
ℹ TDS Meter — 3V3 pin only! Connect VCC to the ESP32 3V3 pin (left power rail, orange wire). Do NOT connect to VIN/5V — the TDS meter is designed for 3.3V and may give wrong readings or be damaged at 5V.
✅ pH Sensor (PH-4502C) Powers from 5V. PO output is ≤ 3.3V for pH range 6-9 (normal water). Connect PO directly to GPIO35 — no voltage divider needed for typical use. Calibrate: dip probe in pH-7 buffer, adjust trim pot.
✅ Relay Module IN connects directly to GPIO26. Relay is set to active-LOW in firmware (RELAY_ACTIVE_HIGH = false). If pump behavior is reversed, change to true in the firmware config section.
ℹ WiFi — use 2.4GHz ESP32 only supports 2.4GHz networks. Use network HG8145V5_43963 (NOT the _5G version). Credentials are entered via the mobile app BLE provisioning.
